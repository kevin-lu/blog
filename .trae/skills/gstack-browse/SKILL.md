## 核心指令

你现在有了浏览器的控制权。可以像用户一样操作浏览器，验证 UI 行为。

**核心能力**：
- 导航到任意 URL
- 点击元素
- 填写表单
- 截图
- 读取控制台错误
- 等待页面加载
- 执行 JavaScript

## 操作方法

### 1. 启动开发服务器

首先确保开发服务器运行：

```bash
# 检查是否有服务器运行
curl -s http://localhost:3000 > /dev/null && echo "Server running" || echo "Server not running"

# 如果没运行，启动服务器
npm run dev
# 或
npm run dev -- -p 3000
```

### 2. 基本操作

#### 导航
```
打开 http://localhost:3000
访问 /posts
去到关于页面
```

#### 点击
```
点击登录按钮
点击第一个文章卡片
点击导航栏的"关于"
```

#### 表单
```
在搜索框输入"Vue"
在评论框输入"很棒的文章"
填写用户名: admin
填写密码: ******
提交登录表单
```

#### 等待
```
等待页面加载完成
等待 2 秒
等待元素 #loading 消失
```

#### 截图
```
截取当前页面
截取 #app 区域
```

#### 读取状态
```
获取当前 URL
获取页面标题
读取控制台错误
检查登录状态
```

### 3. 验证场景

#### A. 页面加载验证
```
1. 访问页面
2. 等待加载
3. 检查标题
4. 截图
5. 检查控制台错误
```

#### B. 用户流程验证
```
1. 访问首页
2. 点击文章卡片
3. 验证文章详情页加载
4. 截图
```

#### C. 表单验证
```
1. 访问表单页面
2. 填写字段
3. 提交
4. 验证成功/错误提示
```

#### D. 响应式验证
```
1. 设置视口为 375x667 (手机)
2. 截图
3. 验证移动端布局
4. 设置视口为 1920x1080 (桌面)
5. 截图
6. 验证桌面布局
```

## 触发条件

- 用户要求视觉验证
- UI 实现后检查
- 端到端测试
- 使用 "/browse" 命令

## 限制

- 需要开发服务器运行
- 需要无头浏览器环境（如 Playwright/Puppeteer）
- 无法测试需要真实支付的流程
- 某些认证流程可能需要特殊处理
