# AI 智能改写功能使用指南

## 📖 功能介绍

AI 智能改写功能允许您输入微信公众号文章链接，系统会自动抓取文章内容并使用 MiniMax AI 进行改写，生成原创内容并发布到博客。

### 核心特性

- ✅ 自动抓取微信文章
- ✅ AI 智能改写，避免版权问题
- ✅ 多种改写策略可选
- ✅ 支持多种文章模板
- ✅ 自动发布或草稿箱模式
- ✅ 实时任务进度监控
- ✅ 成本透明，自动记录

---

## 🔧 配置步骤

### 1. 获取 MiniMax API Key

1. 访问 [MiniMax 开放平台](https://platform.minimaxi.com/)
2. 注册/登录账号
3. 进入「控制台」→「API Keys」
4. 创建新的 API Key
5. 复制 API Key 备用

### 2. 配置环境变量

在项目根目录创建 `.env` 文件：

```bash
# MiniMax AI 配置
MINIMAX_API_KEY=your_minimax_api_key
MINIMAX_MODEL=M2.7
```

> ⚠️ **注意**: 不要将 `.env` 文件提交到 Git！

### 3. 重启服务

```bash
# Nuxt.js 后端
cd app
npm run dev

# 前端
cd frontend
npm run dev
```

## 📝 使用指南

### 1. 访问 AI 改写页面

登录管理后台后，访问：
```
http://localhost:5174/admin/ai-generator
```

### 2. 填写改写参数

#### 参考文章链接
- 输入微信公众号文章链接
- 示例：`https://mp.weixin.qq.com/s/xxx`

#### 改写策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **标准改写** | 保留核心观点和结构，重新组织语言 | 技术教程、概念讲解 |
| **深度改写** | 添加案例分析、扩展说明 | 深度文章、最佳实践 |
| **创意改写** | 完全重新创作，仅保留核心思想 | 观点类文章、评论 |

#### 文章模板

| 模板 | 说明 | 结构特点 |
|------|------|----------|
| **教程类** | 步骤式教学 | 引言 → 步骤 1 → 步骤 2 → 总结 |
| **概念类** | 概念讲解 | 定义 → 原理 → 示例 → 应用 |
| **对比类** | 多方案对比 | 问题 → 方案 A → 方案 B → 对比 → 推荐 |
| **实战类** | 项目实战 | 背景 → 需求 → 实现 → 优化 |

#### 发布设置

- **保存到草稿箱**（推荐）：AI 改写后保存为草稿，人工审核后发布
- **立即发布**：AI 改写后直接发布到前台

### 3. 开始改写

点击「🚀 开始改写」按钮，系统会：

1. 创建任务记录
2. 异步执行抓取和改写
3. 实时显示进度
4. 完成后通知

### 4. 查看进度

在「改写进度」卡片中查看：
- 任务状态（等待/处理中/完成/失败）
- Token 消耗
- 成本估算
- 文章链接

### 5. 查看结果

改写完成后：
- 点击「查看文章」查看前台效果
- 点击「编辑草稿」进行人工润色

## 💰 成本说明

### 计费方式

MiniMax API 按 Token 计费：
- M2.7 模型：¥0.002 / K tokens（输入）
- M2.7 模型：¥0.008 / K tokens（输出）

### 预估成本

| 文章长度 | 输入 Token | 输出 Token | 预估成本 |
|----------|-----------|-----------|----------|
| 短文（1000 字） | ~500 | ~800 | ¥0.007 |
| 中篇（3000 字） | ~1500 | ~2400 | ¥0.022 |
| 长文（5000 字） | ~2500 | ~4000 | ¥0.037 |

> 💡 **提示**: 实际成本取决于原文长度和改写策略

### 每日预算建议

- 每天 3-5 篇文章
- 预估日成本：¥0.10 - ¥0.20
- 预估月成本：¥3 - ¥6

## 🎯 最佳实践

### 1. 选择合适的改写策略

**技术教程** → 标准改写
- 保留技术细节
- 重新组织语言
- 添加代码注释

**最佳实践** → 深度改写
- 保留核心观点

**观点评论** → 创意改写
- 仅保留核心思想
- 完全重新创作
- 加入个人见解

### 2. 人工审核要点

即使选择「自动发布」，也建议人工检查：

- ✅ 技术术语准确性
- ✅ 代码示例正确性
- ✅ 逻辑连贯性
- ✅ 事实准确性
- ✅ 敏感内容过滤

### 3. 避免版权问题

- ✅ 保留核心思想，完全重写表达
- ✅ 添加个人理解和案例
- ✅ 标注参考来源（可选）
- ❌ 不要直接复制原文
- ❌ 不要仅做同义词替换

## 🔍 故障排查

### 问题 1：任务一直显示「等待处理」

**原因**: 后端服务未启动或 API 路径错误

**解决**:
```bash
# 检查后端服务
cd app
npm run dev

# 检查 API 是否正常
curl http://localhost:3000/api/admin/articles/ai-rewrite
```

### 问题 2：改写失败，显示「抓取失败」

**原因**: 微信文章链接无效或需要登录

**解决**:
- 确保链接是公开的
- 确保链接格式正确
- 尝试在浏览器中打开链接

### 问题 3：显示「API Key 无效」

**原因**: 环境变量未正确配置

**解决**:
```bash
# 检查 .env 文件
cat .env

# 重启服务
npm run dev
```

### 问题 4：成本过高

**原因**: 文章过长或改写策略过于复杂

**解决**:
- 选择「标准改写」策略
- 限制文章长度（建议 < 5000 字）
- 设置每日预算上限
