# AI 文章改写优化说明

## 改进概述

针对用户反馈的"改写太简要，看不出内容重要性"和"图片减少"两个核心问题，进行了以下优化：

---

## 1. 内容细化优化

### 1.1 Prompt 优化

**Standard 策略（标准改写）：**
- ✅ 要求"详细列出所有关键观点（不要遗漏任何重要观点）"
- ✅ 要求"提取所有技术细节、代码示例、配置参数"
- ✅ 要求"提取文章中的所有案例、场景、数据"
- ✅ 明确禁止"不要过度简化内容"、"不要省略技术细节"
- ✅ 字数要求：至少 3000-5000 字

**Deep 策略（深度改写）：**
- ✅ 要求"详细讲解技术原理和实现细节"
- ✅ 要求"保留所有优缺点分析和对比"
- ✅ 要求"每个技术点都要深入展开，不要浅尝辄止"
- ✅ 字数要求：4000-6000 字

**Creative 策略（创意改写）：**
- ✅ 要求"保留所有技术细节、代码示例、配置参数"
- ✅ 要求"文风可以轻松，但技术内容必须详细深入"
- ✅ 字数要求：3000-5000 字

### 1.2 禁止事项

所有策略都明确禁止：
- ❌ 不要简化技术内容
- ❌ 不要省略关键细节
- ❌ 不要删除代码示例
- ❌ 不要合并或压缩段落

---

## 2. 图片处理优化

### 2.1 图片提取功能

新增 `extract_images_from_html()` 函数：
- 从原文 HTML 中提取所有图片
- 提取图片的 `src` 和 `alt` 属性
- 过滤掉图标、表情等小图片
- 返回图片列表供后续处理

### 2.2 图片保留策略

**在 Markdown 转换时：**
1. 在文章开头添加"文章图片说明"章节
2. 列出所有原文图片的 URL 和说明
3. 提示 AI 改写时保留图片或用 Mermaid 图表替代

**HTML 转 Markdown 改进：**
- 保留图片的 alt 文本：`![alt 文本](图片 URL)`
- 如果没有 alt 文本，添加提示：`![](图片地址：URL)`
- 确保所有图片都被保留在 Markdown 中

### 2.3 数据流增强

**抓取阶段：**
```python
scraped = {
    'title': '文章标题',
    'content': '文章内容',
    'images': [  # 新增图片列表
        {'src': '图片 URL', 'alt': '图片说明'},
        ...
    ]
}
```

**Markdown 构建：**
```markdown
# 文章标题

## 原文图片
**图片 1**: 图片说明
![图片说明](图片 URL)

## 文章内容
...
```

### 3.2 关键代码变更

**图片提取：**
```python
def extract_images_from_html(html: str) -> list[Dict[str, str]]:
    images = []
    img_pattern = r'<img[^>]+src=["\']([^"\']+)["\'][^>]*(?:alt=["\']([^"\']*)["\'])?[^>]*>'
    
    for match in re.finditer(img_pattern, html, re.IGNORECASE):
        src = match.group(1)
        alt = match.group(2) or ''
        
        # 跳过图标、表情等
        if any(skip in src.lower() for skip in ['emoji', 'icon', 'gif', 'spacer', 'blank']):
            continue
        
        images.append({'src': src, 'alt': alt, 'type': 'image'})
    
    return images
```

**Markdown 转换中的图片处理：**
```python
# 首先提取所有图片并添加说明
images = extract_images_from_html(html)
if images:
    img_section = "## 文章图片说明\n\n"
    img_section += "> 原文包含以下图片，改写时请保留这些图片或在相应位置使用 Mermaid 图表/文字描述替代\n\n"
    for i, img in enumerate(images, 1):
        img_section += f"![图片{i}: {img['alt'] or '无说明'}]({img['src']})\n"
        img_section += f"> 位置：{img['src'][:80]}...\n\n"
    img_section += "---\n\n"
    markdown = img_section + markdown
```

## 4. 使用建议

### 4.1 选择合适的改写策略

- **技术文章** → 使用 `deep` 策略，保留深度和细节
- **教程类文章** → 使用 `standard` 策略，平衡可读性和细节
- **观点类文章** → 使用 `creative` 策略，保持技术细节的同时增加个人风格

### 4.2 人工审核

AI 改写后建议进行人工审核：
1. 检查技术细节是否完整保留
2. 验证代码示例是否正确
3. 确认图片是否保留或适当转换
4. 添加个人见解和案例

## 5. 预期效果

### 改写前 vs 改写后

**改写前（问题）：**
- ❌ 内容过于简要，丢失技术深度
- ❌ 图片被忽略或减少
- ❌ 代码示例被简化

**改写后（目标）：**
- ✅ 保留所有技术细节和深度
- ✅ 所有图片都被保留或转换
- ✅ 代码示例完整保留
- ✅ 字数充实（3000-6000 字）
- ✅ 表达自然但内容详细
