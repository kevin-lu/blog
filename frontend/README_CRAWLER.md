# 自动文章抓取系统 - 前端使用指南

## 📱 访问管理界面

### 1. 启动前端

```bash
cd frontend
npm install
npm run dev
```

### 2. 访问管理后台

打开浏览器访问：`http://localhost:5173/admin`

登录后，在左侧菜单中找到 **"自动抓取"** 菜单项。

---

## 🎯 功能说明

### 仪表盘概览

进入自动抓取管理页面后，您可以看到：

- **RSS 源数量**：已配置的 RSS 源总数
- **待处理队列**：等待 AI 改写的文章数量
- **已完成改写**：已成功改写的文章数量
- **定时任务**：正在运行的定时任务数量

### RSS 源管理

在 **RSS 源管理** 卡片中：

- 查看已配置的所有 RSS 源
- 查看每个源的详细信息（URL、类别、抓取限制）
- 通过开关启用/禁用 RSS 源
- 支持按类别筛选

**支持的 RSS 源**：
- 博客园 (cnblogs)
- V2EX (v2ex)
- Solidot (solidot)
- 掘金 (juejin) - 需代理
- 美团技术 (meituan) - 需代理
- 知乎日报 (zhihu_daily) - 需代理

### 定时任务状态

在 **定时任务状态** 卡片中：

- 查看所有定时任务的运行状态
- 查看下次执行时间
- 查看上次执行时间
- 手动触发任务执行
- 启动/停止定时任务

**定时任务列表**：
- **每日文章抓取** - 每天凌晨 2:00 执行
- **AI 队列处理** - 每天凌晨 2:30 执行
- **历史数据清理** - 每周日凌晨 3:00 执行

### AI 队列处理进度

在 **AI 队列处理进度** 卡片中：

- 查看待处理、处理中、已完成、失败的任务数量
- 点击 **"查看队列详情"** 查看详细列表
- 点击 **"手动处理队列"** 立即处理积压任务

**队列详情** 支持：
- 按状态筛选（待处理/处理中/已完成/失败）
- 分页查看
- 重试失败任务

### 抓取历史

在 **抓取历史** 表格中：

- 查看每次抓取的详细记录
- 查看抓取源、发现数量、新增数量、重复数量、错误数量
- 按时间排序

### 手动抓取

点击右上角的 **"手动抓取"** 按钮：

1. 选择要抓取的 RSS 源（可多选）
2. 设置抓取数量（1-100）
3. 点击 **"开始抓取"**

系统会立即执行抓取任务，并将新文章加入 AI 改写队列。

## 🔧 常用操作

### 操作 1: 立即抓取文章

1. 点击右上角 **"手动抓取"** 按钮
2. 选择 RSS 源（如：cnblogs, v2ex, solidot）
3. 设置抓取数量（如：20 篇）
4. 点击 **"开始抓取"**

### 操作 2: 查看 AI 队列状态

1. 点击 **"查看队列详情"** 按钮
2. 切换到不同标签页查看各状态的队列项
3. 对于失败的任务，点击 **"重试"** 按钮重新处理

### 操作 3: 手动处理队列

如果队列积压较多文章：

1. 点击 **"手动处理队列"** 按钮
2. 系统会立即开始处理待处理的文章
3. 可以在队列详情中查看处理进度

### 操作 4: 管理定时任务

**启用/禁用定时任务**：
1. 找到对应的定时任务
2. 点击 **"启动"** 或 **"停止"** 按钮

**手动触发定时任务**：
1. 找到对应的定时任务
2. 点击 **"执行"** 按钮
3. 任务会立即执行一次

**查看任务日志**：
- 在定时任务卡片中查看上次和下次执行时间
- 刷新页面查看最新状态

### 操作 5: 管理 RSS 源

**启用/禁用 RSS 源**：
1. 找到对应的 RSS 源
2. 点击开关切换启用状态

**建议**：
- 如果网络环境限制，可以禁用需要代理的源（掘金、美团技术、知乎日报）
- 只启用稳定的源（博客园、V2EX、Solidot）

## 💡 使用技巧

### 技巧 1: 批量处理

如果队列中积压了大量文章：
1. 先点击 **"手动处理队列"** 开始处理
2. 系统会串行处理，避免触发 API 限流
3. 可以关闭页面，后台会继续处理

### 技巧 2: 监控告警

如果配置了告警通知（钉钉/企业微信）：
- 队列积压超过阈值会自动发送告警
- AI 改写失败会发送告警
- RSS 抓取失败会发送告警

### 技巧 3: 定时任务优化

建议配置：
- 每日文章抓取：每天 2:00 执行
- AI 队列处理：每天 2:30 执行（抓取完成后）
- 历史数据清理：每周日 3:00 执行

### 技巧 4: 性能优化

- 使用批量操作（已实现）
- 定期清理历史数据
- 监控队列长度，避免积压

## ❓ 常见问题

### Q1: 手动抓取后没有看到新文章？

**A**: 可能的原因：
1. 所有文章都已抓取过（去重机制）
2. RSS 源暂时无法访问
3. 网络问题

**解决方案**：
- 查看抓取历史，确认是否有新增
- 检查 RSS 源是否启用
- 稍后重试

### Q2: AI 队列处理很慢？

**A**: 这是正常现象：
- 系统使用串行处理，避免触发 API 限流
- 每篇文章约需 3-5 分钟
- 可以查看队列详情了解处理进度

**解决方案**：
- 耐心等待，后台会自动处理
- 可以配置多个时间段处理

### Q3: 定时任务没有执行？

**A**: 可能的原因：
1. 定时任务被禁用
2. 服务器时间不正确
3. 应用未启动

**解决方案**：
- 检查定时任务状态
- 手动触发一次
- 查看应用日志

### Q4: 如何查看详细的错误信息？

**A**: 
1. 在队列详情中查看失败任务的错误信息
2. 查看后端应用日志
3. 查看定时任务执行日志
